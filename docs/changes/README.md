# Change Notes

Use this directory for implementation-facing change records.

Create one file per notable change.

Recommended filename:
- YYYY-MM-DD-short-topic.md

Include:
- summary
- affected files
- affected environment
- operational impact
- compatibility notes
- follow-up work

# Change Notes

Use this directory for durable implementation-facing change notes.

Keep entries when they explain a current repository behavior or source-of-truth decision.

Do not use this directory as a phase-by-phase work log once the change has been folded into the canonical docs.
