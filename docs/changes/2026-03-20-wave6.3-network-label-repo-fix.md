# Wave 6.3 Network Label Repo Fix

Date: 2026-03-20
Scope: production compose network-key alignment for the existing production network
Change type: compose metadata consistency fix

## What Changed

- changed the logical network key in `infra/docker-compose.prod.yml` from `corapan-prod` to `corapan-network-prod`
- updated all service network references in the same file to use the new logical key
- kept the real network name unchanged as `name: corapan-network-prod`
- documented the forensic root cause and static verification in `docs/state/Welle_6.3_network_label_repo_fix_summary.md`

## Why It Changed

Production forensics showed that Compose V2 and the existing production network are already present and in active use. The remaining conflict is between the logical Compose network key stored in the repository and the legacy compose-network label attached to the live network.

Aligning the logical key to the live production reality is the smallest safe repo-only fix because it avoids network recreation, avoids `external: true`, and preserves the existing real network name.

- infra/docker-compose.prod.yml

- no cleanup or server-side intervention is introduced
