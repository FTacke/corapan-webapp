# 2026-03-24 CI Repair

Scope:
- Root-Level GitHub Actions unter `CORAPAN/.github`
- App-Code und Tests unter `app/`

Geaendert:
- CI von deaktivierten Tests auf ehrliche Gates umgestellt
- Fullsuite-Hash-Matrix ersetzt durch fokussierte Hash-Kompatibilitaetsabdeckung
- Postgres-Smoke und Playwright auf kanonischen Auth-Pfad ausgerichtet
- Importzeit-Konfigurationsabhaengigkeiten reduziert
- gemeinsame SQLAlchemy-Query-Defekte in Auth-/Admin-Flows behoben

Warum:
- die bisherige CI war rot oder fachlich entwertet
- mehrere Root Causes lagen nicht in GitHub Actions selbst, sondern in App-Importen und ORM-Filtern

Auswirkung:
- schnellere und ehrlichere Pflicht-CI
- klare Trennung zwischen servicefreien und servicegebundenen Checks
- `bcrypt` bleibt als Kompatibilitaetspfad abgesichert, ohne doppelte Vollsuite

Kompatibilitaet:
- Runtime-Policy bleibt streng: `AUTH_DATABASE_URL` und `BLS_CORPUS` bleiben Pflichtwerte bei echter App-Nutzung
- historische SQLite-E2E-Helfer werden in der CI nicht mehr als kanonischer Pfad genutzt
